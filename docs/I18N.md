# Internationalization (i18n)

DuoduoManager supports English (`en`) and Simplified Chinese (`zh-Hans`). The language is determined by the macOS system language setting — there is no in-app language switcher.

## How it works

### Resource files

Localization strings are stored in `.lproj` directories:

```
Sources/Resources/
├── en.lproj/
│   ├── Localizable.strings    # UI strings
│   └── InfoPlist.strings      # Info.plist strings (privacy descriptions)
└── zh-Hans.lproj/
    ├── Localizable.strings    # UI strings
    └── InfoPlist.strings      # Info.plist strings
```

### Type-safe access

All strings are accessed through `L10n.swift`, which provides a nested enum structure:

```swift
// Usage in views:
Text(L10n.Status.running)           // "Running" / "运行中"
Text(L10n.DaemonConfig.title)       // "Daemon Settings" / "Daemon 配置"
Button(L10n.Config.save) { ... }    // "Save" / "保存"

// With interpolation:
L10n.Error.executionFailed("timeout")  // "Execution failed: timeout" / "执行失败: timeout"
```

### Bundle resolution

`L10n.bundle` automatically resolves the correct bundle:

- **In `.app` bundle**: Uses `Bundle.main` (`.lproj` directories are in `Contents/Resources/`)
- **In `swift run`**: Uses `Bundle.module` (SPM-processed resources)

### String key format

Keys use dot-separated hierarchical names:

- `status.running` — Top-level namespace
- `daemonConfig.listenPort` — Feature-specific namespace
- `error.executionFailed %@` — With printf-style interpolation

## Adding a new language

1. Create a new `.lproj` directory under `Sources/Resources/`:

```bash
mkdir Sources/Resources/ja.lproj
```

2. Copy and translate `Localizable.strings` and `InfoPlist.strings`

3. Update `Package.swift` to include the new resource:

```swift
resources: [
    .process("Resources/en.lproj"),
    .process("Resources/zh-Hans.lproj"),
    .process("Resources/ja.lproj"),  // <-- add this
],
```

4. Update `build_app.sh` — the wildcard `cp -r Sources/Resources/*.lproj` will automatically pick up the new directory

5. Update `Info.plist` `CFBundleLocalizations` array to include the new language code

## Adding a new string

1. Add the key to both `en.lproj/Localizable.strings` and `zh-Hans.lproj/Localizable.strings`

2. Add a static property to `L10n.swift` in the appropriate namespace

For strings with interpolation, use `String(localized:)` with string interpolation syntax:

```swift
// .strings file:
"error.custom %@" = "Custom error: %@";

// L10n.swift:
static func custom(_ detail: String) -> String {
    String(localized: "error.custom \(detail)", bundle: bundle)
}
```
